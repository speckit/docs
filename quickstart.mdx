# Get Started with SpecKit in 5 Minutes

Welcome to **spec-driven development**! This quickstart guide will have you creating executable specifications with **SpecKit** methodology in under 5 minutes.

<Note>
**What You'll Learn**: How to write your first executable specification, generate an implementation plan, and experience the transformation from documentation to working code.
</Note>

## Prerequisites

Before we begin, make sure you have:

- **Python 3.11+**
- **uv** package manager
- **Git** for version control
- A **text editor** (VS Code recommended)
- **5 minutes** of focused time

## Step 1: Install Spec Kit

The Spec Kit templates are installed via the `specify` CLI using `uvx`:

<Note>
**Prerequisites**: You need Python 3.11+, Git, and [uv](https://docs.astral.sh/uv/) package manager installed.
</Note>

### For New Projects

```bash
# Create a new SpecKit project
uvx --from git+https://github.com/github/spec-kit.git specify init my-first-spec-project
```

### For Existing Projects

```bash
# Add SpecKit to current directory
uvx --from git+https://github.com/github/spec-kit.git specify init --here
```

<Tip>
**Pro Tip**: Use `specify check` to verify local tooling. The `/specify`, `/plan`, and `/tasks` commands are surfaced by your AI coding agent.
</Tip>

### Choose Script Type (optional)

All automation scripts have both Bash (`.sh`) and PowerShell (`.ps1`) variants. You can force one with `--script`:

```bash
uvx --from git+https://github.com/github/spec-kit.git specify init my-project --script sh  # POSIX shells
uvx --from git+https://github.com/github/spec-kit.git specify init my-project --script ps  # PowerShell
```

## Step 2: Configure AI Agent

The **SpecKit** initialization will prompt you to select your AI coding agent. You can also specify it directly:

    Requires [Claude Code](https://www.anthropic.com/claude-code) to be installed.
  </Tab>
  
  <Tab title="GitHub Copilot">

    ```
    
    Works with [GitHub Copilot](https://code.visualstudio.com/) in VS Code.
  </Tab>
  
  <Tab title="Cursor">

    ```
    
    Integrates with [Cursor](https://cursor.sh/) AI coding environment.

This creates the essential Spec Kit structure:

```
my-first-spec-project/
├── .specify/
│   ├── scripts/
│   └── templates/
├── specs/
└── CONSTITUTION.md
```

## Step 3: Use the /specify Command

Open your AI coding agent (Claude Code, Cursor, etc.) in the project directory. You should see the `/specify`, `/plan`, and `/tasks` commands available.

Use the **`/specify`** command to create your first specification:

<Tabs>
  <Tab title="Claude Code Example">
    ```text
    /specify Build a user registration system that allows new users to create accounts with email and password. The system should validate email formats, enforce password complexity rules, prevent duplicate email addresses, and send confirmation emails after successful registration.
    ```
    
    Claude Code will automatically:
    1. Create a new branch (e.g., `001-user-registration`)
    2. Generate `specs/001-user-registration/spec.md`
    3. Include user stories and functional requirements
  </Tab>
  
  <Tab title="Manual Specification">
    If not using an AI agent, create the specification manually:
    
    ```markdown
    # User Registration Feature
    
    ## User Stories
    - As a new user, I want to create an account with email and password
    - As a user, I want my password to be secure and validated
    - As a system, I want to prevent duplicate email registrations
    
    ## Functional Requirements
    - **FR-001**: System MUST validate email format
    - **FR-002**: System MUST enforce password complexity rules
    - **FR-003**: System MUST prevent duplicate email addresses
    - **FR-004**: System MUST send confirmation email after registration
    
    ## Acceptance Criteria
    - Valid email formats are accepted
    - Passwords must be 8+ characters with mixed case and numbers
    - Duplicate emails return appropriate error message
    - Confirmation email is sent within 30 seconds
    ```
  </Tab>
</Tabs>

## Step 4: Generate Implementation Plan

Use the **`/plan`** command to create a technical implementation plan:

```text
/plan The application uses Node.js with Express framework, PostgreSQL database for user storage, bcrypt for password hashing, and SendGrid for email delivery. The frontend should be a simple HTML/CSS/JavaScript interface with form validation.
```

This will automatically generate:

- **Technical architecture** decisions
- **Implementation phases** and milestones  
- **Technology recommendations**
- **Risk assessment** and mitigation strategies

<Accordion title="Example Generated Plan">
```markdown
# Implementation Plan: User Registration Feature

## Technical Architecture
- **Backend**: Node.js with Express
- **Database**: PostgreSQL with user table
- **Validation**: Joi schema validation
- **Email**: SendGrid integration
- **Security**: bcrypt password hashing

## Implementation Phases
### Phase 1: Core Registration (Week 1)
- Database schema creation
- User model implementation
- Basic registration endpoint

### Phase 2: Validation & Security (Week 1)
- Email format validation
- Password complexity rules
- Duplicate email prevention

### Phase 3: Email Integration (Week 2)
- SendGrid configuration
- Confirmation email templates
- Email sending workflow
```
</Accordion>

## Step 5: Generate Task Breakdown

Use the **`/tasks`** command to break down the plan into actionable tasks:

```text
/tasks
```

This will automatically create `specs/001-user-registration/tasks.md` with specific, assignable tasks:

<Steps>
  <Step title="Database Setup">
    - Create user table schema
    - Set up database migrations
    - Configure connection pooling
  </Step>
  
  <Step title="API Implementation">
    - Implement POST /register endpoint
    - Add input validation middleware
    - Create user service layer
  </Step>
  
  <Step title="Security Implementation">
    - Add password hashing
    - Implement rate limiting
    - Add CSRF protection
  </Step>
  
  <Step title="Email Integration">
    - Configure SendGrid API
    - Create email templates
    - Implement confirmation workflow
  </Step>
</Steps>

## Common First Questions

<Tabs>
  <Tab title="How is this different?">
    **Traditional development** translates requirements into code manually, introducing errors and drift.
    
    **SpecKit** transforms specifications into implementation automatically, maintaining perfect alignment.
  </Tab>
  
  <Tab title="What about existing projects?">
    You can adopt **SpecKit** incrementally:
    1. Start with new features using specifications
    2. Gradually add constitutional governance
    3. Introduce automation over time
  </Tab>
  
  <Tab title="Does it replace developers?">
    No! **Spec Kit** amplifies developer capabilities by:
    - Eliminating repetitive planning tasks
    - Reducing documentation maintenance
    - Preventing specification drift
    - Allowing focus on creative problem-solving
  </Tab>

## Troubleshooting

<AccordionGroup>
  <Accordion title="uvx command not found">
    **Issue**: `uvx` command is not available.
    
    **Solution**: Install uv package manager first:
    ```bash
    # macOS/Linux
    curl -LsSf https://astral.sh/uv/install.sh | sh
    
    # Windows (PowerShell)
    powershell -c "irm https://astral.sh/uv/install.ps1 | iex"
    ```
  </Accordion>
  
  <Accordion title="AI agent commands not available">
    **Issue**: `/specify`, `/plan`, `/tasks` commands not showing in AI agent.
    
    **Solution**: Ensure you're in the correct directory and the AI agent is properly configured:
    ```bash
    # Check if spec-kit files exist
    ls -la .specify/
    ls -la CLAUDE.md  # or your agent's file
    
    # Restart your AI coding agent
    ```
  </Accordion>
  
  <Accordion title="Python version issues">
    **Issue**: Python 3.11+ is required but not available.
    
    **Solution**: Install the correct Python version:
    ```bash
    # macOS (Homebrew)
    brew install python@3.11
    
    # Ubuntu/Debian
    sudo apt install python3.11
    
    # Verify version
    python3 --version
    ```
  </Accordion>
</AccordionGroup>
